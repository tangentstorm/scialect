#+title: scialect proof agent

* overview

This is a tool for organizing parallel work on large proofs.

* current (manual) algorithm

- I run 5 agents under tmux (jc0 .. jc4).
  "jc" stands for "jacobian challenge", the formalization I'm working on.
- agent 0 is the "manager". the others are workers
- each worker has its own working copy of the repo under ~/ver (ex: ~/ver/jc2)
- each worker has its own branch at any given moment
- managers and workers communicate via two files in each worker's directory:
  - goal.md is the current goal for the worker
  - result.md (if present) indicates that the worker has stopped, either because
    the work is finished or because it needs help
- the manager creates the goal file according to a detailed prompting guide.
  the goal file includes instructions like:
  - rebase on origin/main and build the solution
  - do your work on some existing branch, or create a new one
  - don't take shortcuts
  - prove `theorem someTheoremName` which is usually currently `by sorry`
  - follow this particular proof strategy...
  - if you can't prove the statement, explain what's blocking you
  - commit your changes (with a co-authored-by line)
  - when you are done, summarize what you did in result.md
- generally speaking, the agent gets a new session for each new goal.
  currently, i just manually restart it, and add a prompt like:
  ```
  /goal follow the instructions in goal.md
  ```
  (may change slightly based on which agent is doing the work)
- whenever a worker stops done, I tell the manager about it; the manager inspects the worker output in result.md and writes review feedback to review.md.
  then, it decides whether the work is ready to merge into the project, or whether the prompt
  needs adjustment.
- the (intended) benefit of this is that the manager's context window stays populated with
  the big picture view of the project.
